PARTY UI RULESET v1（LLM-READY / BILINGUAL）

Use English for model parsing. Chinese annotations included for human readability.

⸻

1. Core Identity / 核心定位

EN (Rule):
Party is a neutral civic interface for all possible events.
The platform must remain calm, precise, structural, and low-noise.
Event content provides emotion, color, and personality; the platform never competes with user-generated visuals.

CN（註釋）:
Party 是「承載所有活動的中立公共界面」。平台保持冷靜、精準、有結構、低噪音。
所有情緒與色彩都由活動本身提供，平台不與內容搶視覺主權。

⸻

2. Visual Language Composition / 視覺語彙比例

EN (Rule):
All UI decisions must simultaneously reflect the following four styles, equally weighted (25% each):

 1. Calm Tech (minimal, precise, low-noise)
 2. Human Note (light warmth, readable spacing rhythm)
 3. Civic Interface (public wayfinding clarity)
 4. Swiss Structure (grid-first, hierarchy-first)

CN（註釋）:
四種語彙等權重：冷靜科技、人文氣息、城市導覽、瑞士結構。
每一項設計輸出都需同時符合這四種語言。

⸻

3. Structure–Emotion Ratio / 結構與情緒比例（70/30）

EN (Rule):
70% of UI must be structural (layout, grid, hierarchy).
30% of UI may contain emotional cues (user-provided hero image or video).
The ratio cannot be reversed under any circumstance.

CN（註釋）:
70% 結構＋30% 情緒。活動素材只能支配 30%。不可反轉。

⸻

4. Color System / 顏色規則

EN (Rule):
Platform colors = Neutral only (White/Off-White, Light Gray, Dark Gray).
One functional accent color is allowed (static, non-emotional).
Event visuals define their own colors; platform must not impose color identity.
Gradients, saturated colors, or decorative color blocks are forbidden in platform UI.

CN（註釋）:
平台＝中性色（白灰黑）＋一個固定的功能性強調色。
活動才有色彩。平台禁止大面積色塊與高飽和色。

⸻

5. Typography / 字體與排版規則

EN (Rule):
Use sans-serif fonts (Inter, Helvetica, Noto Sans).
Left-aligned text only.
Line-height: 1.4–1.6.
Hierarchy defined strictly by size/weight/spacing (not decoration).
Standard scale:
 • H1 32px
 • H2 26px
 • H3 20px
 • Body 16px

CN（註釋）:
全部左對齊、無襯線體、層級依大小/粗細/間距決定。

⸻

6. Card Architecture / 活動卡片語法

EN (Rule):
Card = fixed structural template.
Hero media (image/video) may occupy up to 30% visual weight.
Content hierarchy (mandatory order):

 1. Title
 2. Time
 3. Location
 4. Tags
 5. Host summary
Host weight = medium (not dominant).
No irregular card shapes, no heavy shadows, no gradient UI backgrounds.

CN（註釋）:
卡片是固定結構：標題→時間→地點→標籤→主辦。
Hero 媒體最多 30%。不允許不規則形狀或裝飾。

⸻

7. Tag System Semantics / 標籤語義規則

EN (Rule):
Tag design must follow this semantic blend:
 • 40% transit-wayfinding clarity
 • 30% GitHub topic neutrality
 • 20% Douban humanistic softness
 • 10% Airbnb category lightweight visuals
Tags represent semantic meaning, not decoration.

CN（註釋）:
標籤＝地鐵導引＋技術中性＋豆瓣柔和＋Airbnb 輕量。
標籤屬語意，不屬裝飾。

⸻

8. Host Profile Rules / 主辦語法

EN (Rule):
Host identity must be defined by:
A. Professional capability (events hosted)
C. Behavioral data (history, reliability, participation graph)
Do not use personality-based or social-network style presentation.
Host page = Event CV, not a social profile.

CN（註釋）:
Host 以活動履歷與行為資料定義，不做人設、不社交化，頁面像履歷。

⸻

9. Real Human Definition / Real Human 定義

EN (Rule):
“Real human” must be expressed through behavioral traces, not portraits.
Examples: participation history, event graph, host credibility, attendance patterns.

CN（註釋）:
Real human＝行為痕跡，而非頭像。

⸻

10. Motion Rules / 動效規則

EN (Rule):
Motion must be minimal and functional:
 • fade, slide, opacity transitions (0.12–0.2s)
 • no bounce, no playful elasticity
Hero video allowed but must not disrupt layout or readability.

CN（註釋）:
動效極簡、功能性。影片需 overlay 與固定結構。

⸻

11. Tone & Copywriting / 文案語氣

EN (Rule):
Tone must be neutral, concise, civic, and non-social.
No emotional prompts, no exclamation marks, no playful language.

CN（註釋）:
文案＝中立、簡潔、城市公共語氣。避免社交式熱情語。

⸻

12. Strict Prohibitions / 嚴禁項目

EN (Rule):
The platform UI must not contain:
 • strong brand color dominance
 • decorative gradient backgrounds
 • playful UI elements
 • excessive motion
 • content-first emotional bias
 • social-network aesthetic
 • color theming that overrides event content

CN（註釋）:
平台 UI 嚴禁：品牌色淹沒內容、裝飾漸層、可愛語氣、重動效、社交風格、奪走活動主導權。

⸻

13. Platform Role Definition / 平台角色定義

EN (Rule):
The platform is a container, not a storyteller.
The event is the protagonist; the platform is the structural frame.
All LLM-generated UI must reflect this relationship.

CN（註釋）:
平台＝容器，活動＝主角。任何 LLM 輸出都必須遵守此結構關係。
